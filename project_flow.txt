## ✅ (Wikibase Cortex Assistant)**

### **Project Name:**  
`wikibase-cortex-assistant`

---

### **🧩 Problem:**  
Users need natural-language access to structured knowledge (e.g., Wikipedia content) without searching or reading pages manually.

---

### **🎯 Goal:**  
Build a **question-answering assistant** using Snowflake Cortex that retrieves relevant content from a vector database and generates accurate answers.

---

### **📥 Data Collection Plan:**

- Fetch articles from **Wikipedia** (or use public Wikipedia dumps).
- **Chunk** long articles into segments (300–500 words each).
- Store chunks in **Snowflake**.
- Use **Snowflake Cortex Embedding** to vectorize the chunks.

---

### **💡 RAG Flow:**

1. **User sends a question** to FastAPI.
2. FastAPI app uses **Cortex Embedding API** to embed the query.
3. Query vector is used to perform **vector search in Snowflake** (top-k chunks).
4. Retrieved chunks + question sent to **Cortex LLM**.
5. Response is returned to the user via FastAPI.

---

### **🧠 Model Used:**
- **Snowflake Cortex Embedding**
- **Snowflake Cortex LLM (command or mixtral)**

---

### **⚙️ Approach:**
- Retrieval-Augmented Generation (RAG)
- Context window: top-k relevant chunks (e.g., top 3)

---

### **🚀 Tech Stack:**
- **FastAPI** (API layer)
- **Snowflake** (data store + Cortex)
- **Python**
- **Conda** (env management)
- **CI/CD**: GitHub Actions
- **Deployment**: AWS (EC2 or Lambda)
